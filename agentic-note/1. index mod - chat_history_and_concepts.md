## 🛠️ Explanations of Technologies & Concepts Used

### 1. The Experimental "HTML in Canvas" API (WICG)
Normally, `<canvas>` elements contain only flat pixels. The proposed API allows combining DOM layout mechanics with canvas drawing contexts:

* **`layoutsubtree`**: Tells the browser's layout engine to compute styling, positions, and accessibility for elements inside the `<canvas>` tags even though they are not visible on the main page.
* **`drawElementImage(element, x, y)`**: Captures the rendered pixels of the subtree element and draws them directly into the canvas.
* **`paint` Event**: Fires whenever a canvas child's state (hover, focus, edit) changes, triggering a redraw without polling.

> [!WARNING]
> Because it is experimental and requires a manual flag change (`chrome://flags/#canvas-draw-element`), it should **not** be used in user-facing production sites.

---

### 2. Chromatic Aberration & Glitch Transitions (CSS)
To create a high-fidelity glitch animation that matches the theme's style, we used keyframe-based CSS rules:

* **Opacity Vanishing:** The text is set to `opacity: 0` during the middle phase of the keyframe timeline (`30%` to `45%`). This creates a complete "vanish" moment where the text content is secretly swapped behind the scenes using JavaScript.
* **Chromatic Aberration:** During the entry and exit phases of the glitch (`15%`, `60%`, `75%`), we apply horizontal translations (`transform: translate()`) coupled with offsets in the `text-shadow` property:
  ```css
  text-shadow: 2px -1px 0 #ff00c1, -2px 1px 0 #00fff0;
  ```
  This overlays a red/magenta and cyan copy of the text shifted by a few pixels, simulating digital screen interference.

---

### 3. State Persistence via Cookies (JavaScript)
To ensure the transition is a special "first-visit" experience, we use HTTP cookies via the browser's `document.cookie` API:

* **Cookie Retrieval (`getCookie`)**: Uses a regular expression to match and extract the value of `gaus_website_effect_run` from the header cookie string.
* **Cookie Storage (`setCookie`)**: Sets `gaus_website_effect_run=true` with a `max-age` set to 365 days (`SameSite=Lax` for security), ensuring that reload or subsequent visits bypass the script.

```javascript
// Example Cookie Check
if (!getCookie('gaus_website_effect_run')) {
  // Execute effect...
  setCookie('gaus_website_effect_run', 'true', 365);
}
```

---

### 4. DOM Lifecycle & Asynchronous Timing
The script utilizes nested timing chains via `setTimeout()` to execute actions in a exact sequence:

1. **Page Load Check**: Waits for `DOMContentLoaded` so all target headers are available in the DOM.
2. **First Delay (3000ms)**: Allows the visitor to settle on the page for 3 seconds before initiating the transition.
3. **Glitch Activation**: Adds the `.glitch-active` CSS class to start the 600ms animation.
4. **Content Swap (220ms)**: Executes exactly during the vanish window, replacing the `textContent` and `innerHTML` of the header items.
5. **Class Cleanup (600ms)**: Removes the `.glitch-active` class once the visual transition completes.
6. **Revert (10000ms)**: After 10 seconds of displaying the modified layout, the timeline repeats in reverse to seamlessly restore the original text.
