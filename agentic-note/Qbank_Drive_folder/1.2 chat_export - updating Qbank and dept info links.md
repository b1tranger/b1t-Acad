## Detailed Technical Q&A / Discussion

### Q1: Why were you trying to restore `index.html` from Git history?

During the initial replacement to populate the live QBank nested browser in `index.html`, a closing tag mismatch (an extra `</div>`) was introduced. When attempting to remove this extra closing tag, the regex/string matching patterns in the file modification tool matched incorrectly across similarly structured nested blocks, leading to the accidental truncation/deletion of critical parts of the QBank section (including the Open Full QBank link and surrounding tags).

Since `git` was not globally available in the system PATH, we ran search scripts to locate the portable git executable installed by GitHub Desktop under AppData. Once found, we ran `git checkout 9b68607 -- index.html` to revert the file to its clean pre-edit state. We then applied the nested browser changes and verified tag alignment with zero errors.

### Q2: What python scripts were created to fetch and process Google Drive contents?

To avoid manually copying dozens of links, folder IDs, and files from Google Drive, we created several utility Python scripts in the scratch directory to parse the metadata programmatically:

- **`fetch_subfolders.py`:** Fetched the public HTML structure of the Google Drive folder IDs for the major departments (CSE, CE, IT, BBA, Law, Others).
- **`parse_drive_structure.py` / `parse_subfolders.py`:** Extracted details from the fetched Google Drive HTML files, parsing individual subfolder shortcut IDs, document viewing links, and course names, mapping them to a structured JSON format (`qbank_folders.json`).
- **`parse_dept_info.py`:** Parsed the department info links shortcut folder to retrieve the corresponding drive folder shortcut links for each department, which we then added to `departments-data.js`.
These scripts allowed us to build an accurate, comprehensive QBank dropdown database structure and department schema dynamically and reliably.
