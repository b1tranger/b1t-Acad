# Implementation Plan - Support Selected Semester in URL

Allow users to share direct links to specific semesters within a department by supporting a `sem` query parameter (e.g. `?dept=CSE&sem=5`).

## User Review Required

> [!NOTE]
> We will update the browser URL using `history.replaceState` when a user selects a semester. This will show the selected semester (e.g. `&sem=5`) in the address bar without reloading the page, making it instantly copyable and shareable.

## Proposed Changes

### Department Selection Logic

#### [MODIFY] [departments.js](file:///c:/Users/gsmur/Documents/GitHub/%5BoU1TS%5D/b1t-Acad/js/departments.js)

- **Initial Load Parsing**:
  Modify the `DOMContentLoaded` event listener to check for the `sem` parameter in the URL.
  Implement a flexible parser that supports format variations such as `sem=5`, `sem=05`, and `sem=S05`.
  If a valid semester is found in the query parameter, load it on page load; otherwise, fall back to the first semester of the department as default.

- **Dynamic URL Updating**:
  Modify the `loadSemester` helper function to update the browser's address bar query parameters using `window.history.replaceState`.
  This will set/update `dept` and `sem` values automatically when a semester is selected in the UI.

5. Try navigating to a department with no semesters (e.g. `BBA`) and verify `sem` is not appended to the URL.
