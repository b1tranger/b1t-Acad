# Walkthrough - Semester URL Parameter Support

I have upgraded the department page logic to support loading and sharing specific semesters via the `sem` URL query parameter (e.g. `?dept=CSE&sem=5`).

## Changes Made

### Department Selection Logic
- **[departments.js](file:///c:/Users/gsmur/Documents/GitHub/%5BoU1TS%5D/b1t-Acad/js/departments.js)**:
  - Added logic in `DOMContentLoaded` to parse the `sem` parameter from the URL query string.
  - Handled flexible inputs (e.g. `sem=5`, `sem=05`, `sem=S05`, `sem=s5`).
  - Modified the `loadSemester` helper function to update the browser URL search parameters using `window.history.replaceState` when a new semester is active.

### Documentation
- **[DOCUMENTATION.md](file:///c:/Users/gsmur/Documents/GitHub/%5BoU1TS%5D/b1t-Acad/doc/DOCUMENTATION.md)**:
  - Updated key feature description for **Dynamic Routing** to document semester parameter support.
  - Added semester URL update and routing capabilities description to `js/departments.js` module entry.
  - Documented version `6.1` updates in the Version History table.

## Verification Results

A browser subagent verified all requirements:
1. Checked default page load on `http://127.0.0.1:3000/Departments.html?dept=CSE`.
2. Verified that selecting "Semester 5" dynamically updates the URL in the address bar to `?dept=CSE&sem=5` without reloading the page.
3. Navigated directly to `?dept=CSE&sem=5` and verified that Semester 5 loads immediately on load.
4. Navigated to `?dept=CSE&sem=s5` and verified that it correctly resolved to Semester 5 content and updated the URL parameter to `5`.
