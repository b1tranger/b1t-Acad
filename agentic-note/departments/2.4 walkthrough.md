# Walkthrough - Departments Page Layout & Tooltip Optimizations

I have implemented layout adjustments, responsive design optimizations, and scroll-triggered bottom tooltip alerts for the Departments page:

1. **Relocated "Open Full Drive" button**: Moved from the header of the semester content section to sit directly under the "Course Materials" heading.
2. **Scroll-Triggered Tooltip**: Added a viewport-fixed tooltip displaying `refer to "Others" folder for Note Archive` that triggers after "Course Materials" has been in view for 3 seconds, stays visible for exactly 3 seconds, and then fades out.
3. **Responsive Side-by-Side Buttons**: Aligned the "Departmental Info" and "Question Bank" buttons side-by-side using Flexbox. On mobile viewports (<= 480px width), the labels automatically contract to "Info" and "Qbank" to save space and fit horizontally.

---

## Changes Made

### Styling & CSS rules
- **[Departments.html](file:///c:/Users/gsmur/Documents/GitHub/%5BoU1TS%5D/b1t-Acad/Departments.html)**:
  - Added CSS rule `.dept-buttons-row` to wrap the departmental actions in a flex container that divides space evenly when both buttons are active.
  - Implemented `@media (max-width: 480px)` overrides to hide the desktop text (`.btn-text-desktop`) and reveal mobile text (`.btn-text-mobile`).
  - Added fixed viewport styling for `.bottom-tooltip` with smooth opacity and transform transitions.
  - Placed the placeholder `<div id="bottom-tooltip" class="bottom-tooltip">` at the end of the HTML structure.

### Rendering & Scroll Logic
- **[departments.js](file:///c:/Users/gsmur/Documents/GitHub/%5BoU1TS%5D/b1t-Acad/js/departments.js)**:
  - Updated button innerHTML rendering for both `info` and `qbank` buttons to structure them using desktop and mobile text spans.
  - Modified the HTML structure in the `loadSemester` helper function to place the Google Drive link directly under the "Course Materials" header.
  - Configured `IntersectionObserver` on the `#courses` section to track visibility. When visible, it starts a 3-second delay, adds the `.show` class to the tooltip, delays another 3 seconds, and removes it.
