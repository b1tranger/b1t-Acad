## 2. Problems Faced & Solutions

| Problem Faced | Root Cause | Solution Implemented |
| :--- | :--- | :--- |
| **`404 Not Found` on gviz/tq query** | Google Visualization API (`/gviz/tq`) cannot parse/execute on public-published `2PACX-` URLs. | Migrated fetch links to direct public CSV exports (`output=csv`) and wrote a custom lightweight client-side CSV parser. |
| **Parsing Errors on Cell Commas/Line Breaks** | Standard `.split(',')` or `.split('\n')` fails when student submissions contain commas, quotes, or multiline comments. | Developed a robust `parseCSV(text)` helper that handles double quotes, nested commas, and native `\r\n` / `\n` line endings. |
| **Google Drive "Access Denied"** | Google Form file upload folders default to **Restricted** access permissions. | Documented step-by-step instructions for sharing the GDrive upload folder as *"Anyone with the link can view (Viewer)"* so links resolve globally. |

## 3. Detailed Script Analysis

Here is a breakdown of the core script logic developed and used to test and handle submissions.

### A. The Client-Side CSV Parser (`parseCSV`)
This function iterates character-by-character to parse CSV contents correctly according to RFC 4180 standards.

```javascript
function parseCSV(str) {
    const arr = [];
    let quote = false;
    let col = 0, row = 0;

    for (let c = 0; c < str.length; c++) {
        let cc = str[c], nc = str[c + 1];
        arr[row] = arr[row] || [];
        arr[row][col] = arr[row][col] || '';

        // Double quotes inside a quoted field (escaped quote "")
        if (cc == '"' && quote && nc == '"') {
            arr[row][col] += cc; ++c; continue;
        }
        // Toggle quote state
        if (cc == '"') {
            quote = !quote; continue;
        }
        // Split columns by comma only if we're not inside quotes
        if (cc == ',' && !quote) {
            ++col; continue;
        }
        // Split rows by Carriage Return + Line Feed (\r\n) outside quotes
        if (cc == '\r' && nc == '\n' && !quote) {
            ++row; col = 0; ++c; continue;
        }
        // Split rows by single Line Feed (\n) outside quotes
        if (cc == '\n' && !quote) {
            ++row; col = 0; continue;
        }
        // Split rows by single Carriage Return (\r) outside quotes
        if (cc == '\r' && !quote) {
            ++row; col = 0; continue;
        }
        // Build cell content character by character
        arr[row][col] += cc;
    }
    return arr;
}
```

### B. Submissions Processing Block (`fetchQuestions` and `fetchNotes`)
These functions fetch their respective Google Sheet CSV file, parse it using `parseCSV`, skip the header row, and group the submissions by student name.

```javascript
// Example from fetchNotes:
const rows = parseCSV(text);
const submitterMap = new Map();

for (let i = 1; i < rows.length; i++) {
    const row = rows[i];
    if (!row || row.length <= N_COL_NAME) continue;

    const name = String(row[N_COL_NAME] || '').trim();
    if (!name || name === '' || name.toLowerCase() === 'student name') continue;

    const email = String(row[N_COL_EMAIL] || '').trim();
    const studentId = String(row[N_COL_STUDENT_ID] || '').trim();
    const deptBatch = String(row[N_COL_DEPT_BATCH] || '').trim();
    const details = String(row[N_COL_DETAILS] || '').trim();
    const filesRaw = String(row[N_COL_FILES] || '').trim();

    // extract GDrive URLs from cell
    const files = parseFileLinks(filesRaw);

    const submission = { deptBatch, details, files, timestamp };

    // Grouping by student name
    if (submitterMap.has(name)) {
        const submitter = submitterMap.get(name);
        submitter.count++;
        submitter.submissions.push(submission);
    } else {
        submitterMap.set(name, {
            name, email, studentId, count: 1, submissions: [submission]
        });
    }
}
```
